# Implementation Traceability Matrix

This document is the **single source of truth** for implementation status against project-plan requirements.

## Status Legend

- `implemented`: fully present in the current codebase **and** backed by at least one automated test reference.
- `partial`: requirement exists but is incomplete, indirect, or lacks direct end-to-end validation.
- `missing`: requirement is not present in the current codebase.

## Release Gate Rule

Release is **BLOCKED** if any row is `partial`/`missing` or if the test reference column is empty/non-automated.

## Maintenance Rules (Single Source of Truth)

1. Any new requirement must be added here before implementation starts.
2. Any merged feature must update `Current status`, evidence links, and test reference in the same PR.
3. A requirement cannot move to `implemented` without at least one explicit automated test reference proving the requirement.
4. If code and this matrix disagree, update this matrix in the same PR that resolves the mismatch.
